#

零 DP 内存优化

这是《零：训练一万亿个参数模型的内存优化》一文中介绍的零 DP 的实现，

它将优化器状态、梯度和参数的分片保存到多个设备/节点中。它减少了原始模型Nd​(2+2+K)Ψ​的内存消耗，其中Ψ是参数的数量，Nd​是分片的数量，K是每个参数的优化器字节数。2+2是假设精度为 16 位的参数和梯度存储器；即每个参数和梯度为 2 个字节。K=12对于 Adam 优化器，因为它维护参数的副本，在 fp32 中每个参数两个时刻。

零 DP 的通信量为O(3Ψ)。比较而言，数据并行训练的通信量为O(2Ψ)。

尽管它被命名了Zero3 ，但我们只实现了其中的零 DP 部分，没有实现针对剩余内存消耗的 Zero-R 内存优化。Out 实现仅支持训练一部分参数。

此实施的灵感来自公平规模的财务安全发展计划。

以下是使用零 DP 内存优化微调 GPT NeoX 的脚本。
