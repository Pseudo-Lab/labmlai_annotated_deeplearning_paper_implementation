#

Zero-DP 메모리 최적화

이 문서는 ZeRO: Memory Optimization Towards Training A Trillion Parameter Models에서 소개된 Zero-dP를 구현한 것입니다.

옵티마이저 상태, 그래디언트, 파라미터들을 여러 장치 및 노드들에 저장합니다. 이 모델은 원래 모델의 Nd​(2+2+K)Ψ​ 만큼 메모리 풋프린트를 줄입니다. 이때 Ψ는 모델의 파라미터 수입니다. Nd​는 샤드의 수입니다. 그리고 K 파라미터마다 옵티마이저 최적화 바이트 수입니다. 2+2는 16비트 정밀도를 가정한 파라미터와 그래디언트 메모리입니다; 즉, 파라미터당 그래디언트가 2입니다. K=12는 Adam 최적화 기법이 파라미터 복사본과 fp32에서 매개변수당 두 개의 모멘트를 유지하기 때문입니다.

제로-DP의 통신량은 O(3Ψ)입니다. 비교를 위해 데이터 병렬 훈련의 통신량은 O(2Ψ)입니다.

제로3라고 명명되었지만, 잔여 메모리 소비를 목표로 하는 제로-R 메모리 최적화는 구현하지 않고 제로-DP 부분만 구현했습니다. 아웃 구현은 일부 파라미터의 하위 집합만 훈련을 지원합니다.

이 구현은 Fairscale FSDP에서 영감을 받았습니다.

다음은 ero-DP 메모리 최적화를 사용하여 GPT-NeoX를 파인튜닝하는 스크립트입니다.
